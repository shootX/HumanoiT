# Taskly CRM

CRM სისტემა პროექტების, ამოცანების, აქტივების და ბიზნეს პროცესების მართვისთვის.

## ტექნოლოგიები

- **Backend:** Laravel, PHP
- **Frontend:** React, Inertia.js, TypeScript, Tailwind CSS
- **ბაზა:** MySQL

## ფუნქციონალი

- **პროექტები** – შექმნა, რედაქტირება, ბიუჯეტი, მილსტოუნები
- **კონტაქტები** – ფიზიკური და იურიდიული პირები, Excel import (ქართული)
- **ამოცანები** – Task stages, კომენტარები, ჩეკლისტები
- **აქტივები** – აქტივების მართვა, კატეგორიები, Excel export/import (ქართული)
- **Bugs** – ბაგების მიკვლევა, სტატუსები
- **Timesheets** – დროის აღრიცხვა
- **ინვოისები** – ინვოისების მართვა
- **კომპანიები და Workspaces** – მრავალკომპანიური რეჟიმი

## ვერსიების ისტორია

### v1.5.2 (2026-02-25)

**აქტივების ლოგიკა დავალებებში**
- დავალების ფორმაში მხოლოდ ხელმისაწვდომი (active) აქტივები ჩანს
- გამოყენებული (used) ნივთები სელექტში არ ჩანს
- რაოდენობის ჩამოკლება და სტატუსის განახლება გამოყენებისას
- რედაქტირებისას უკვე მინიჭებული used აქტივების შენარჩუნება

**ინვოისი – აქტივის ავტოკომპლეტი**
- 3+ სიმბოლოს აკრეფისას ჩნდება არსებული აქტივების ჩამონათვალი
- არჩევისას ნივთი უკავშირდება არსებულ asset-ს (დუბლიკატების თავიდან აცილება)
- ძებნა name და asset_code ველებით

**აქტივის ნახვა – შეძენის ინვოისები**
- ჩანს ყველა ინვოისის ჩამონათვალი, რომლიდანაც მოხდა შეძენა
- თითოეული ინვოისი ლინკით, რაოდენობით და თარიღით

### v1.5.1 (2026-02-24)

**დავალებების Bulk Actions**
- მონიშვნა – checkbox-ები table, card და kanban ვიუებში
- "Select all" – ყველა დავალების მონიშვნა table ვიუაში
- Bulk სტატუსის შეცვლა – მონიშნულ დავალებებზე სტატუსის ერთდროული შეცვლა
- Bulk მინიჭების შეცვლა – რამდენიმე პირის მინიჭება ერთდროულად (1, 2 ან მეტი)
- Bulk წაშლა – მონიშნული დავალებების წაშლა დადასტურებით

**მინიჭების ფილტრი**
- კომპანია, მომხმარებელი და კლიენტი – სიაში ჩანს workspace წევრები და პროექტის კლიენტები
- მრავალი არჩევანი – SimpleMultiSelect-ით რამდენიმე შემსრულებლის ფილტრაცია
- ტიპის ნიშნები – სახელი (კომპანია), სახელი (წევრი), სახელი (კლიენტი)

**CRM კონტაქტები**
- იურიდიული პირის შექმნა – `name` ველი ავტომატურად ივსება company_name ან brand_name-ით

### v1.5.0 (2026-02-23)

**შესყიდვების რეპორტი**
- Reports → შესყიდვების რეპორტი – ნივთის დასახელება, რაოდენობა, ერთეულის ფასი, სრული ფასი, დავალება, პროექტი
- მხოლოდ შეძენილი აქტივები (type=asset), მომსახურება გამორიცხულია
- ფილტრები: პროექტი, თარიღი (დაწყება–დასრულება), ძიება
- Excel ექსპორტი

**ინვოისები – რამდენიმე პროექტი და დავალება**
- ინვოისის ფორმაში რამდენიმე პროექტის და დავალების მონიშვნა
- ხარჯის გადანაწილება ფილიალებს შორის – დავალების „გამოყენებული აქტივების“ მიხედვით (asset_task)
- ხელით გადანაწილება ინვოისის ფორმაში ამოღებულია – მონაცემები მოდის მხოლოდ დავალებიდან

### v1.4.0 (2026-02-19)

**აქტივები და ინვოისების ინტეგრაცია**
- Task-ში აქტივების სია – რაოდენობით, ინვოისიდან შექმნილი აქტივები (quantity > 0)
- ამოცანაზე გამოყენებული აქტივების ჩვენება (asset_task pivot)
- ინვოისის რედაქტირება ნებისმიერი სტატუსით
- ინვოისის განახლება – სინქრონიზაცია Asset-ის quantity და task-ის აქტივებთან
- ინვოისის ფორმა – ამოღებულია აქტივის სელექტორი; quantity და rate აბსორბირებენ ათწილადებს
- ინვოისის Index და Show – დამატებულია დაკავშირებული Task
- Assets – მოხსნილია რეზერვის ფუნქცია, დატოვებულია მხოლოდ quantity
